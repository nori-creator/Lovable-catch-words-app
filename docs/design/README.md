## TL;DR — 最重要提言トップ10

1. **復習開始が遅い問題を最優先で直す**: いまは復習を開くたびにカード1枚につきAIを2往復以上呼んで4択の誤答を作っている(`src/lib/reviews.functions.ts` の `getDueReviews`)。誤答は**カード保存時に生成してDBに保存**し、さらに「自分が集めた同カテゴリの単語」から選べばAIコストほぼゼロで教育的にも最良。→ [02 §3.3](./02-core-experience.md)
2. **撮影→意味表示を2〜3秒にする「速報パス」**: 候補5件生成と詳細カード生成を直列で待たせず、まず1コールで「単語+意味+発音」だけ即表示し、詳細はバックグラウンドで生成。ペイン①(調べる手間)の本質解。→ [02 §2](./02-core-experience.md)
3. **「再遭遇」をコアループに昇格させる**: 同じ物をもう一度撮った瞬間は、記憶科学的に最強の復習チャンス。複製カードを作るのではなく「思い出しクイズ→遭遇カウント++→SRS前倒し消化」にする。何回も出会う看板ほど覚えられる、があなたのコンセプトの核。→ [02 §4](./02-core-experience.md)
4. **セキュリティ穴を塞ぐ**: `updateWordExtras`(`src/lib/stickers.functions.ts:298`)が共有 `words` テーブルを所有者チェックなしで更新できる=他人のカード解説を書き換えられる。ストア公開前に必ず修正。→ [03 §1](./03-overlooked.md)
5. **TTS音声をキャッシュする**: 同じ単語の音声を毎回生成している。Supabase Storage に `(言語, テキスト, 声)` キーでmp3を保存して使い回す。復習での音声自動再生(あなたの要望)もこれが前提。→ [02 §3.2](./02-core-experience.md)
6. **PWA化(manifest + Service Worker + Web Push)**: SRS通知・ホーム画面追加・オフライン撮影キュー・「同じ場所を通ったらテスト」の全部がここに乗る。その後 **Capacitor で同じコードのまま App Store / Google Play へ**。→ [04 §4](./04-architecture.md)
7. **無料枠の制限(1日n枚)と使用量テーブルを先に入れる**: 課金導線より先に「AIコストの上限」が要る。制限はそのままPro課金の訴求点になる。→ [05 §3](./05-roadmap-and-business.md)
8. **AIの嘘(ハルシネーション)対策**: 意味・注音は AI 任せにせず、教育部辞書(萌典API)・TOCFL語彙表と突き合わせて「辞書確認済み」バッジを出す。学習アプリで一番信頼を失うのは間違った意味を覚えさせること。→ [03 §4](./03-overlooked.md)
9. **自撮り+位置情報はデフォルト非公開**: ソーシャル機能を伸ばす前に、共有時のEXIF/GPS除去・通報/ブロック(ストア審査の必須要件)を仕様化。→ [03 §2-3](./03-overlooked.md)
10. **Lovable がなくても開発できる状態を作る**: リポジトリはローカルで `bun install && bun dev` で動く構成。AI呼び出しだけが `LOVABLE_API_KEY` に依存しているので、プロバイダ切替レイヤーを1ファイル足せば Gemini直/Claude/任意のOpenAI互換APIに差し替え可能。→ [04 §2-3](./04-architecture.md)

## このドキュメントの使い方

- **Lovable で進める場合**: 各セクションの「仕様」ブロックをそのままプロンプトとして貼る(1機能=1メッセージ推奨)。
- **Claude Code で進める場合**: [04-architecture.md §2](./04-architecture.md) のローカル開発手順に従って環境を作り、ロードマップ([05](./05-roadmap-and-business.md))の項目を1つずつ依頼する。
- どちらで進めても、**DBマイグレーションは必ず `supabase/migrations/` に残す**(両環境の同期が保たれる)。
