## 2. やらないことリスト(いまは)

- アプリ内SNSの深堀り(DM、グループ、公開フィードのアルゴリズム)→ IG輸出で代替
- ネイティブアプリの書き直し(React Native等)→ Capacitor で足りる
- 汎用辞書アプリ化(検索起点の機能拡張)→ 「出会い起点」の軸がぶれる
- 広告モデル → 単価が低く体験を壊す。課金一本

## 3. 収益化

### プラン設計

| | Free | Pro(月額。地域価格: 台湾/日本で調整) |
|---|---|---|
| キャプチャ | 5枚/日 | 無制限 |
| SRS復習 | 無制限(学習の核は無料を貫く) | 無制限 |
| 詳細カード(extras) | 3枚/日はフル、以降は速報のみ | 全カードフル |
| 日記添削 | 3回/週 | 無制限 |
| 発音チェック | 粗い判定のみ | 発音コーチ(声調フィードバック) |
| 週間リキャップ | 透かしあり | 透かしなし+カスタム背景 |
| 図鑑背景・カードテーマ | 基本のみ | 追加テーマ |

原則: **記憶の核(SRS復習・既存カードの閲覧)は絶対に課金壁にしない**。
「思い出を人質に取る」課金は炎上リスクかつ学習アプリとして不誠実。課金は「もっと集めたい・もっと上手くなりたい」の加速に限定する。

### 決済

- Web(PWA): Stripe(Supabase との連携パターンが確立している)。`entitlements` テーブル([04 §6](./04-architecture.md))を webhook で更新
- ストア版: RevenueCat で App Store / Play / Stripe の購読状態を一元化(自前実装は沼)
- 無料枠の制限(N7)を先に入れておくと、課金は「上限の解除」を売るだけになり実装が薄くなる

## 4. ローンチ計画(台湾×日本人)

1. **クローズドβ(いま〜)**: 友達・彼女+在台日本人の知人 10〜20人。招待制にして「招待された感」を演出。毎週1回、使われ方をヒアリング(特に「撮ったのに保存しなかった瞬間」= 速報パスの検証)
2. **コミュニティシード**: 台湾の日本人コミュニティ(Facebookグループ、留学生会、ワーホリ)に「台湾華語が写真で図鑑になる」デモ動画を投下。**週間リキャップのIG共有(X5)がそのまま広告になる**ので、β参加者のIG投稿を最重要KPIにする
3. **公開(PWA)**: ストア審査を待たずにURLだけで配れる。QRコードを語学学校・日本人向けカフェに置く(オフラインの物理接点がこのアプリのコンセプトと合う)
4. **ストア公開**: Capacitor 版。ASOキーワードは「台湾華語」「繁体字」「TOCFL」等のニッチで確実に1位を取れる語から
5. コンテンツマーケ: 「台湾の街で覚えた単語100」系の図鑑スクショ記事/Reels。UGCがそのまま素材になる

## 5. 計測設計

**North Star Metric: 週あたり「定着語数」**(その週に repetitions ≥ 3 に到達した単語数/アクティブユーザー)。
「撮った数」でも「開いた日数」でもなく、学習アプリとしての約束(覚えられる)を直接測る。

主要ファネルとイベント:

```
install → first_capture(60秒以内が目標)
        → capture_saved(速報パス完遂率)
        → d1_return → first_review_session → d7_streak
        → recap_shared(バイラル係数の源)
        → paywall_view → subscribe
```

最初から仕込むイベント(PostHog等、[04 §7](./04-architecture.md)):
- capture_started / suggest_shown(+レイテンシms) / suggestion_accepted(第1候補か手入力か) / capture_saved / capture_abandoned(どのステップで離脱したか)
- review_session_started / review_answered(モード・正誤・応答ms) / review_session_completed
- reencounter_triggered / recalled(正誤)
- journal_generated / journal_corrected / recap_shared
- paywall_view / subscribe / churn

ガードレール指標: 速報パスのp90レイテンシ、AIコスト/ユーザー/日、キャッシュヒット率(words/tts/choices)、通知許可率、通知経由の復帰率。

## 6. 最初の一歩

このドキュメント承認後の推奨着手順は **N1 → N2 → N4 → N3**。
理由: N1/N2 は既存機能の体感を一気に変える(友達に配る前に必須)、N4 はデータが増える前に直すほど安い、N3 が新体験の核。
いずれも Claude Code に「docs/design/02-core-experience.md の §3.3 を実装して」と該当セクションを指定すれば着手できる粒度で書いてある。
