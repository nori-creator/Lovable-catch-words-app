# 01. ビジョンとギャップ分析

## 1. コンセプトの一行定義

> **「調べる」を消して「出会う」を増やす。** 街で見た言葉は写真1枚でカードになり、
> 出会った場所・自分の顔・その日の物語と一緒に記憶される。

重要なのは、このアプリの敵が Anki や Duolingo ではなく **「調べないまま通り過ぎる」という行動**だということ。
毎回の意思決定を「辞書を引くか引かないか」から「シャッターを押すか押さないか」に置き換える。
シャッターは1タップ、しかも押した瞬間に収集報酬が出る——ここが勝ち筋。

### 価値の3層構造

| 層 | 提供価値 | 対応する既存機能 |
|---|---|---|
| 瞬間 | 撮るだけで意味と発音が分かる(辞書の置換) | capture フロー |
| 日 | 今日の収集がアルバム/日記になる(記録の置換) | home アルバム、journal |
| 週〜月 | SRSと再遭遇で長期記憶になる(Ankiの置換) | review、図鑑、地図 |

3層が独立アプリとして成立するレベルの価値を持ちつつ、写真1枚が全層に同時に燃料を供給する。
この「1アクション→3層に波及」構造が競合が真似しにくい部分なので、
新機能を検討するときは常に「これはどの層を強くするか、撮影1回あたりの価値を増やすか」で判断する。

## 2. ターゲットとローンチ戦略

- **初期(いま)**: 台湾在住・台湾滞在中の日本人 × 台湾華語。友達・彼女が実際に使える品質が最初のマイルストーン。
- **中期**: 日本人 × 中国語(簡体字)/英語/韓国語。UI は日本語のまま対象言語を増やす方が、UI多言語化より先。
- **長期**: UI 多言語化して世界の学習者へ。

初期ターゲットが極めて良い理由:
1. 繁体字圏は「文字は読めるのに音と意味が分からない」日本人特有のペインが最大化する市場(漢字が読める分、撮って音を聞く価値が際立つ)
2. 台湾は街頭の文字密度が高く、夜市・看板・メニューと被写体が無限にある
3. 在台日本人コミュニティは小さく密で、口コミが回りやすい(→ [05](./05-roadmap-and-business.md) のローンチ計画)

## 3. ペイン①〜⑨ × 現状実装のギャップ

あなたが挙げたペインを、実際のコードと突き合わせた結果。
「済」は実装済み、「半」は骨格あり、「未」は未実装。

### ① 街中で見るものを毎回調べる手間

| 要素 | 状態 | 現状と残ギャップ |
|---|---|---|
| 写真→AI候補→カード化 | 済 | `capture.tsx` + `ai.functions.ts`。カメラ自動起動も済 |
| 速度 | **未** | 候補生成→カード生成が直列のAI呼び出しで、意味が分かるまで10秒超になりうる。**「2〜3秒で意味と発音だけ先に出す」速報パスが必要**(→ [02 §2](./02-core-experience.md)) |
| オフライン | **未** | 地下鉄・電波の悪い路地で撮れない。撮影だけローカル保存し、電波復帰後に解析する「あとで解析キュー」(→ [03 §6](./03-overlooked.md)) |
| OSレベルの導線 | **未** | PWAのショートカット/共有ターゲット(カメラアプリで撮った写真をCatchwordsに共有→即カード化)。「アプリを開く」ことすら省く(→ [04 §4](./04-architecture.md)) |

### ② ネイティブの実際の使い方が辞書にない

| 要素 | 状態 | 現状と残ギャップ |
|---|---|---|
| コロケーション/語法/場面/雑学 | 済 | `words.extras` jsonb に collocations, usage_note, common_situation, trivia 等を生成・保存済み |
| 品詞で色分けしたチャンク例文 | **未** | あなたの要望(S/V/O/副詞で色分け)。例文を「チャンク配列」として生成する仕様を [02 §6](./02-core-experience.md) に定義 |
| 実コーパス由来の頻度 | **未** | いまはAIの自己申告レベル。TOCFL語彙表・頻度リストとの照合で「本当の頻度・級」を出す(→ [03 §4](./03-overlooked.md)) |
| 実際に使われている記事/動画リンク | **未** | AIにURLを生成させると嘘リンクになるので、検索クエリを組み立てて外部サービス(YouGlish系、Google検索)に飛ばす方式が安全(→ [02 §6](./02-core-experience.md)) |

### ③ Ankiは無機的・4択が毎回同じ・発音と一緒に覚えられない

| 要素 | 状態 | 現状と残ギャップ |
|---|---|---|
| 自分の写真つき4択 | 済 | `review.tsx` はステッカー切り抜き画像を表示 |
| 毎回違うAI生成の誤答 | 半 | 実装済みだが**毎回その場で生成していて遅い&高コスト**。事前生成+自分の収集語からの出題に変更(→ [02 §3.3](./02-core-experience.md)) |
| ネイティブ音声つき復習 | **未** | 復習画面に音声自動再生がない。TTSキャッシュ(→ [02 §3.2](./02-core-experience.md))を前提に、カード表示と同時に再生 |
| 聞く→発音する→意味を選ぶ | **未** | 習熟段階に応じてモードが昇格する「段階式復習」を仕様化(→ [02 §3.1](./02-core-experience.md)) |
| 難易度をユーザーが調整 | **未** | 誤答の紛らわしさレベル設定(設定画面に3段階)。事前生成方式なら「レベル別に3セット持つ」だけで済む |

### ④ 忘却曲線上の位置が見えない・カード一覧が視覚的でない

| 要素 | 状態 | 現状と残ギャップ |
|---|---|---|
| 忘却曲線グラフ | 済 | `ForgettingCurveChart` + `getOverallMemoryStats`(記憶保持率モデルつき) |
| カードの視覚的一覧 | 済 | 図鑑(dex)がカテゴリ別グリッド表示 |
| **図鑑と記憶状態の融合** | **未** | 図鑑の各ステッカーを記憶保持率で色分け(緑=定着/黄=揺らぎ/赤=忘却危機)。「図鑑を開くだけで、どこが弱いか一目で分かる」= Ankiに対する決定的な差別化(→ [02 §5](./02-core-experience.md)) |
| キャラ育成による記憶の物語化 | 未(Later) | 撮った写真からキャラ生成→復習で進化/放置で退化。ロードマップ Later 枠(→ [05](./05-roadmap-and-business.md)) |

### ⑤ ネットの画像では文脈がない。自撮り・場所・時間で記憶を現実に接続

| 要素 | 状態 | 現状と残ギャップ |
|---|---|---|
| 自撮り+物体の2枚撮影 | 済 | capture フローに selfie ステップあり |
| 場所・日時の記録 | 済 | stickers に lat/lng/location_name/taken_at |
| 地図表示 | 済 | map ルート + Google Maps connector |
| **過去に撮った場所を通ったらテスト** | **未** | PWAではバックグラウンド測位ができないので、「アプリ起動時・地図表示時に近傍照合してクイズ発火」から始め、ネイティブ化後にジオフェンス通知(→ [02 §7](./02-core-experience.md)) |

### ⑥ 社会性とゲームによる行動変容

| 要素 | 状態 | 現状と残ギャップ |
|---|---|---|
| フォロー/投稿/いいね/コメント/通知 | 済 | posts, follows, post_likes, post_comments, notifications テーブル+画面 |
| デイリークエスト(あれを撮ってみよう) | 済 | `quests.functions.ts`(AI生成3件/日) |
| ストリーク | **未** | 継続の心理装置として最重要なのに未実装。XPも stats 内の計算のみで永続化されていない(→ [02 §9](./02-core-experience.md)) |
| 友達タグ付け/一緒に撮るとボーナス | 未(Next) | posts の拡張で可能。ただし他人の顔が写る=プライバシー設計が先(→ [03 §2](./03-overlooked.md)) |
| Instagramへの書き出し | **未** | 週間リキャップ画像の自動生成→OSの共有シートでIGストーリーズへ。**アプリ内SNSを育てるより、既存SNSに「作品」を輸出する方が初期は圧倒的に効率的**(→ [02 §8](./02-core-experience.md)) |

### ⑦ 辞書+Anki登録が写真1枚で済む / ARの未来

- 現在の capture フローがまさにこれ。ギャップは①の速度とOS導線。
- ARグラスは Later 枠だが、**「見る→即座に音と意味」という北極星として全設計判断の基準に使う**。例: 速報パスのレイテンシ予算(→ [02 §2](./02-core-experience.md))はARの応答性基準から逆算している。

### ⑧ 覚えられないカードの解説を作り直すのが面倒

| 要素 | 状態 | 現状と残ギャップ |
|---|---|---|
| 解説の再生成API | 半 | `updateWordExtras` はあるが、**項目ごとの再生成UI**がない |
| 覚えられない語の自動検知 | **未** | SRSで3回連続失敗=「リーチ(leech)」として検知→「ニーモニック再生成」「もう一度実物を撮りに行こう」提案(→ [02 §3.4](./02-core-experience.md)) |

### ⑨ (文面が途切れていた項目)

「Ankiの単語のフラッシュカードを母語の意味だけで…」と読めるため、
**「母語→意味の認識だけで覚えた気になり、産出(自分で言える)に至らない」** 問題と解釈した。
→ 段階式復習([02 §3.1](./02-core-experience.md))の「産出ステージ」(音声 or 入力で答える)がこの解。もし意図が違えば教えてほしい。

## 4. 競合との差別化マップ

| | 収集の楽しさ | 現実世界との接続 | 記憶科学 | ソーシャル | 発音 |
|---|---|---|---|---|---|
| Anki | ✕ | ✕ | ◎(SRS) | ✕ | ✕ |
| Duolingo | △(XP/ストリーク) | ✕ | △ | △(リーグ) | △ |
| CapWords系(撮影→単語) | ○ | △(撮影のみ) | △ | ✕ | ✕ |
| Pokémon GO | ◎ | ◎ | — | ○ | — |
| **Catchwords** | ◎(図鑑+ステッカー) | ◎(場所+自撮り+再遭遇) | ◎(SRS+忘却可視化) | ○(アルバム共有) | ○→◎(段階導入) |

**独自ポジション = 「現実世界との接続 × 記憶科学」の掛け算**。ここを両方持つ競合は存在しない。
逆に言うと、単なる「撮影翻訳アプリ」(Google レンズで代替可能)や単なる「かわいいフラッシュカード」に寄ると即座に競合の海に落ちる。
迷ったら「その機能は、現実で出会った記憶を強化するか?」で取捨選択する。

## 5. 機能追加の判断基準(今後のブレ防止)

1. 撮影1回あたりの価値を増やすか(3層構造のどこかを強くするか)
2. 現実世界との接続を強めるか
3. 「調べる/登録する」系の手数を減らすか
4. 毎日開く理由(due復習、クエスト、友達の投稿、ストリーク)を増やすか
5. 上記に当てはまらない機能は、どれほど魅力的でも Later 送り

この基準で見ると、あなたのアイディアのうち **発音チェック・キャラ育成・企業コラボ** は全部「良いが Later」。
**速報パス・再遭遇・復習の音声化・ストリーク・IG輸出** が Now/Next。詳細は [05](./05-roadmap-and-business.md)。
