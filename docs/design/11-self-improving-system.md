# 自己改善システム v1(docs/design/11-self-improving-system.md)— 2026-07-14

> ビジョン(NORI): 「ユーザーが使えば使うほどデータがたまり、アプリ自体も毎日勉強し、
> 自動でチェック・自己改善して、毎日賢くなるアプリ」。
> 本書はその実装済み部分・設計・**やらないこと(法的境界)**を記録する。

## 1. 実装済み: 毎日まわる自己改善ループ

トリガー: **その日最初のスキャン**(サーバー側 fire-and-forget、20時間スロットル)。
ユーザーが1人でも使えば毎日自動で走る。外部cron不要。手動実行は管理画面「自己改善」パネル。

```
ユーザーの利用 ─┬→ ①学習: スキャン/キャッチ結果を共有辞書へ蓄積(source='ai')
                ├→ ②点検: 辞書ランダム監査(1日1回)
                └→ ③観察: 台湾ニュースRSS→独自コーパス(1日1回)
```

### ① 学習(使うほど貯まる)— 2026-07-14実装
detectScan / detectParts / generateCard の結果を dictionary_entries に insert-only 蓄積。
verified行は絶対に上書きしない。次からは辞書ヒット=AI呼び出しゼロ=最速。

### ② 点検(自動ランダムテスト)— lexicon_audits
- 毎日、AI由来エントリ8件+**verified 2件を抜き打ち**でランダム抽出(uuid一様性サンプリング)
- richモデルが台湾教育部基準で注音・拼音・意味を再検証(1コール/日 ≈ 0.5円)
- 自動修正は「AI由来 × 確信度0.85以上」のみ。**verified行は機械が触らず要レビューに積む**(憲法§2-1)
- 全結果は lexicon_audits に記録 → 管理画面に ✅正確 / 🔧自動修正 / ⚠️要レビュー で表示
- 同じ語も母数が小さいうちは数十日周期で再点検が回る(将来: last_checked優先のサンプリングに改良)

### ③ 観察(アプリ自身の毎日の勉強)— corpus_stats
- 台湾ニュースRSS(Google News TW総合/エンタメ+公視)の**見出しだけ**を毎日取得
- 辞書7,158語との最長一致で分かち書きし、**語×日×出現回数だけ**を蓄積
- **原文は一切保存しない** = 頻度という事実の統計のみ → 著作権的に安全
- これが「ネイティブがどの語をどの頻度で使うか」の**独自コーパス**の種。管理画面に「今週の頻出語」表示
- 将来: カードの「頻度・どこで使う」欄にこの実データを表示(AI推定→実測ハイブリッド)

## 2. やらないこと(法的境界)— 重要

| 構想 | 判定 | 理由と代替 |
|---|---|---|
| Threads/TikTok/Instagramのスクレイピング | ❌ 実装しない | 各社利用規約違反(自動収集禁止)。アカウント停止・法的リスク。**代替**: Meta公式のThreads API(アプリ審査制・自分のアカウントで取得できる範囲)、TikTok Research API(研究者向け)。Phase C以降に公式API申請を検討 |
| YouTube字幕の一括取得 | ❌ | 規約制限。**代替**: ユーザー自身が共有シートから単語を送る(実装済みのshare_target)= ユーザー起点なら合法 |
| ニュース記事本文の保存 | ❌ | 著作権。**実装済みの方式**: 見出しから頻度統計のみ(事実の集計は保護対象外) |
| 公開RSSの見出し観察 | ✅ 実装済み | RSSは配信目的の公開フィード。保存するのは頻度のみ |

## 3. 行動データ分析(YouTube/TikTok型)の設計

**データはすでに全部貯まっている**: stickers(何を・どこで(lat/lng, location_name)・いつ(taken_at)・どんな気持ちで(caption))、scan_events(調べたが保存しなかった語)、review_history(記憶の成績)、usage_events(利用パターン)。

活用アルゴリズム(実装順):
1. **時間×場所の復習リマインド**(Phase B・Capacitor後): 「夜市によく行く時間」に通知、その場所で覚えた単語を出題(文脈依存記憶の活用)
2. **パーソナル出題順**: review優先度 = SM-2期日 × 「その語のカテゴリをよく撮るか」×「corpus_statsの実頻度」— よく出会う語を優先的に定着させる
3. **発見フィード**: 「あなたがよく撮る『食べ物』で、まだ持っていない頻出語」を図鑑の???スロットとして提案
4. **インサイト画面**: 「あなたは金曜の夜、士林夜市で食べ物を撮る人」— 自分の学習が地図と時間で見える

プライバシー原則: 分析は本人向けのみ。他人への公開は明示的な投稿だけ。全データはアカウント削除で消える(実装済み)。

## 3.5 v2追記(2026-07-15)

初回リリースの不具合(監査0件・コーパス0行なのにエラーが見えない)を受けて:
- **self_improve_runs 実行ログ**: 全ステップの成否・件数・エラー・フィード別HTTPステータスを記録し管理画面に表示 — 無言の失敗を廃止
- **フィード5本に多重化**(Google News×2・公視・自由時報・中央社)+ ブラウザ風UA
- **共起データ(corpus_pairs)**: 同じ文に現れた語のペアを蓄積 → 「どの単語と一緒に使われるか」ランキング
- **新語発見**: 見出しの辞書未収録連続漢字(2〜4字、2回以上出現)を候補化 → AIが実在検証して定義付きで辞書へ
- **④ AI合成コーパス(source='ai_synth')**: AIが毎日「台湾人が今日書きそうな短文40個」(夜市・MRT・Dcard/Threads風・LINEチャット等をローテーション)を生成し、頻度・共起・**辞書に載らない口語/流行語**を蓄積。ニュースでは拾えないSNS/日常域をカバーする合法な代替。実測(news)とAI生成(ai_synth)はソース列で厳密に区別
- **全音声化**: TTS事前生成を全レベル+自動蓄積された新語(tocfl_level null)に拡大
